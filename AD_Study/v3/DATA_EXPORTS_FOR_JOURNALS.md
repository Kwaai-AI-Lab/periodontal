# Data Exports for Journal Submission

This document describes all data exports generated by the analysis pipeline that are needed for journal reproducibility requirements.

## Summary

All charts and figures in the manuscript can be reproduced from the Excel files exported by the pipeline. No automatic plotting is done during model runs to keep outputs clean for batch processing.

## Primary Data Files for Journals

### 1. Baseline Model Data (Main Pipeline Runs)
**Files:**
- `results/Baseline_Model_PD_25.xlsx` (25% PD prevalence scenario)
- `results/Baseline_Model_PD_50.xlsx` (50% PD prevalence scenario)
- `results/Baseline_Model_PD_75.xlsx` (75% PD prevalence scenario)

**Generated by:** run_full_analysis.py Step 1 (imports from IBM_PD_AD_v3.py)
**Purpose:** Complete model results with all timesteps and metrics for each PD prevalence scenario

**Note:** Each scenario gets its own Excel file to avoid overwriting. These are generated during the main pipeline runs (Step 1).

#### Sheets:
- **Summary**: Complete summary history with all model metrics by timestep
  - Population counts (alive, baseline, entrants, deaths)
  - Dementia stages (cognitively_normal, mild, moderate, severe)
  - Incidence and prevalence
  - QALYs (patient, caregiver, combined)
  - Costs (NHS, informal/caregiver, societal)
  - Living settings distribution

- **YearlyFlows**: Annual flows of QALYs and costs
  - Columns: time_step, calendar_year, year_qalys_patient, year_qalys_caregiver, year_qalys_total, year_costs_nhs, year_costs_informal, year_costs_societal

- **RiskFactorPrevalence**: Prevalence of each risk factor over time
  - Both in general population (risk_prev_alive_*) and dementia population (risk_prev_dementia_*)
  - Includes: periodontal_disease, hypertension, hearing_difficulty, APOE_e4, obesity, depression, diabetes, and others

- **SeverityByYear**: Dementia severity distribution for target years (2025, 2030, 2035, 2040)
  - Columns: calendar_year, mild_cases, moderate_cases, severe_cases

### 2. Figure Generation Complete Results
**Files:**
- `results/Figure_Generation_PD_25.xlsx` (25% PD scenario - full model results)
- `results/Figure_Generation_PD_50.xlsx` (50% PD scenario - full model results)
- `results/Figure_Generation_PD_75.xlsx` (75% PD scenario - full model results)

**Generated by:** generate_figures_2_3_4_from_model.py (Step 7)
**Purpose:** Complete model results for the figure generation runs
**Note:** These are separate from Step 1 runs and contain the exact data used to create the manuscript figures.

### 3. Manuscript Figures Source Data
**File:** `AD_Model_v3/images_regenerated_model/figure_2_3_4_data.xlsx`
**Generated by:** generate_figures_2_3_4_from_model.py
**Purpose:** Extracted and formatted data specifically for reproducing manuscript Figures 2, 3, and 4

#### Sheets:
- **figure_2_risk_landscape**: Risk factor prevalence data for Figure 2
  - Columns: risk_factor, prevalence_general (%), prevalence_dementia (%), modifiable (yes/no)
  - Year: 2040 (baseline 50% PD scenario)
  - Shows prevalence in general 65+ population vs dementia population

- **figure_3_annual_costs**: Annual societal costs by scenario for Figure 3
  - Columns: calendar_year, scenario (25% PD / 50% PD / 75% PD), total_costs_societal
  - Years: 2024-2040
  - Used to plot annual costs over time for three PD prevalence scenarios

- **figure_4_qaly_differences**: Cumulative QALY differences vs baseline for Figure 4
  - Columns: calendar_year, scenario (25% PD / 75% PD), cumulative_qaly_difference
  - Years: 2024-2040
  - Shows cumulative QALYs gained (25% PD) or lost (75% PD) relative to 50% baseline
  - Negative values = QALY loss, positive values = QALY gain

### 4. PSA Results (Probabilistic Sensitivity Analysis)
**Files:** Multiple Excel files in working directory
**Generated by:** run_psa_direct_v3.py
**Purpose:** Uncertainty analysis with 500 iterations per scenario

#### Files:
- `PSA_2_QALY_Results_PD_XX.xlsx` - QALY outcomes with 95% CIs
- `PSA_2_Cost_Results_PD_XX.xlsx` - Cost outcomes with 95% CIs
- `PSA_2_Incidence_Results_PD_XX.xlsx` - Incidence outcomes with 95% CIs
- `PSA_2_Prevalence_Results_PD_XX.xlsx` - Prevalence outcomes with 95% CIs

Where XX = 25, 50, or 75 (PD prevalence %)

Each file includes:
- Mean, median, SD, 2.5th percentile, 97.5th percentile
- 95% confidence intervals for all outcomes
- All 500 individual draws (for reviewers who want raw data)

### 5. Tornado Analysis (One-Way Sensitivity)
**File:** `pd_sensitivity_analysis.xlsx`
**Generated by:** run_pd_tornado.py
**Purpose:** One-way sensitivity analysis for PD onset hazard ratio

**Note:** Tornado plots are not automatically generated (rendering issues). Use the Excel data to create tornado diagrams manually.

#### Sheets:
- **Baseline**: Baseline scenario results (PD HR = 1.21)
- **Low**: Low PD hazard ratio scenario (HR = 1.07, 95% CI lower bound)
- **High**: High PD hazard ratio scenario (HR = 1.38, 95% CI upper bound)
- **Summary**: Swing analysis showing parameter impact on outcomes

Each sheet includes:
- Total QALYs (patient, caregiver, combined)
- Total costs (NHS, informal, societal)
- Incident dementia cases
- Mean results across replicates with standard errors

#### Creating Tornado Plots from Excel:
1. Open `pd_sensitivity_analysis.xlsx`
2. For each metric (QALYs, costs, incidence):
   - Calculate swing = High value - Low value
   - Plot as horizontal bar chart
   - Reference line at baseline value

### 8. Counterfactual Analysis
**File:** `results/non_preventable_risk_analysis.xlsx`
**Generated by:** run_non_preventable_risk_analysis.py
**Purpose:** Quantify disease burden not attributable to preventable risk factors

#### Sheets:
- **Baseline**: Results with all risk factors present
- **Counterfactual**: Results with all preventable risks removed (genetic APOE-e4 only)
- **Summary**: Comparison showing preventable vs non-preventable burden

## File Naming Convention

To avoid overwriting, all Excel exports now use unique filenames:

**Main model runs (Step 1):** `Baseline_Model_PD_XX.xlsx` where XX = 25, 50, or 75
**Figure generation (Step 7):** `Figure_Generation_PD_XX.xlsx` where XX = 25, 50, or 75

Both sets of files contain the same sheet structure, but:
- **Baseline_Model_PD_XX.xlsx**: Generated first in the pipeline with the configured population
- **Figure_Generation_PD_XX.xlsx**: Generated when creating figures (may use different population if testing)

For journal submission, use either set (they're equivalent if run with the same population). The Figure_Generation files are guaranteed to match exactly what was used to create the manuscript figures.

## Using These Files for Journal Submission

### For Methods Sections:
- Cite the Excel files as supplementary material
- Reference `Baseline_Model_PD_50.xlsx` for complete model parameters and results
- Reference validation CSV files for calibration statistics (R², slopes)

### For Results Sections:
- Use `figure_2_3_4_data.xlsx` to report exact values in figures
- Use PSA Excel files to report 95% confidence intervals
- Use tornado analysis file to report sensitivity ranges

### For Supplementary Materials:
- Include all Excel files as downloadable supplementary data files
- Include this document (DATA_EXPORTS_FOR_JOURNALS.md) as a data dictionary
- Include README_FULL_ANALYSIS.md for reproducibility instructions

### For Reviewer Responses:
- All Excel files contain raw data that reviewers can independently analyze
- PSA files include all 500 individual draws if reviewers want to verify calculations
- Validation files include all observed vs predicted comparisons with error metrics

## Important Notes

1. **No Plots Generated During Model Runs**: The main IBM_PD_AD_v3.py script does NOT generate plots automatically when called from the pipeline. This is intentional to keep batch processing clean. All plotting is done by the figure generation scripts.

2. **Unique Filenames Prevent Overwriting**: Each PD prevalence scenario saves to its own Excel file (`_PD_25`, `_PD_50`, `_PD_75`). This ensures no data is lost when running multiple scenarios.

3. **Data Completeness**: Every Excel file contains ALL data needed to reproduce the corresponding charts. Nothing is omitted or aggregated beyond what's in the files.

3. **Timesteps vs Calendar Years**: The model uses timesteps internally but exports calendar_year for readability. Base year is 2023 (timestep 0 = 2023, timestep 1 = 2024, etc.).

4. **Population Scaling**: PSA and tornado analyses use 1% population for computational efficiency. Results are not scaled back up (they represent outcomes for the 1% sample). Only the main model runs and figure generation use 100% population.

5. **File Locations**: Some files are in the working directory, others in subdirectories. Check the paths above for each file.
